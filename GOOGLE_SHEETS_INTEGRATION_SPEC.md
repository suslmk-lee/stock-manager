# Google Sheets 연동 명세서 (배당금 월별 입력)

## 1) 목적
- 앱에서 배당금을 `생성/수정/삭제`할 때, Google Sheet의 월별 컬럼(`1월~12월`)에만 값을 반영한다.
- 우측 붉은색 컬럼(합계/환산/계좌별합산)은 기존 수식 유지가 원칙이며, 연동 로직에서 직접 쓰지 않는다.

## 2) 시트 컬럼 규칙
- 헤더 기준(1행):
  - `A`: 증권사
  - `B`: 계좌
  - `C`: 대상
  - `D`: 종목
  - `E`: 티커
  - `F`: 현재가
  - `G`: 평가금액
  - `H`: 배당비
  - `I`: 입금일
  - `J`: 수량
  - `K~V`: `1월~12월` (연동 쓰기 대상)
  - `W`: 합계 (수식)
  - `X`: 합계(KRW) (수식)
  - `Y`: 투자금 (수식/수동)
  - `Z`: 계좌별합산 (수식)

## 3) 쓰기 정책 (핵심)
- 연동은 반드시 `K~V` 범위만 업데이트한다.
- `W~Z` 및 그 우측 집계 셀은 절대 쓰지 않는다.
- 월별 값은 "누적합" 방식으로 처리한다.
  - 생성: 기존 월 셀 값 + 배당금
  - 삭제: 기존 월 셀 값 - 배당금
  - 수정: 이전 값/위치 차감 후 새 값/위치 가산

## 4) 행 매칭 키
- 기본 키: `계좌(B) + 티커(E)`
- 보조 키: 필요 시 `종목(D)`까지 확인
- 동일 티커가 여러 계좌에 존재할 수 있으므로, 티커 단독 매칭 금지

## 5) 월 컬럼 매핑
- 배당 `입금일(date)`의 월 기준으로 컬럼 결정
  - 1월 -> `K`
  - 2월 -> `L`
  - ...
  - 12월 -> `V`
- 구현은 고정 인덱스보다 "헤더 텍스트(`1월`...`12월`) 검색" 방식 권장

## 6) 이벤트별 반영 로직
- `CREATE_DIVIDEND`
  1. 행 조회 (`계좌+티커`)
  2. 입금일 월 컬럼 확인
  3. 해당 월 셀에 금액 가산
- `UPDATE_DIVIDEND`
  1. 이전 데이터 기준 월 셀 차감
  2. 신규 데이터 기준 월 셀 가산
  3. 계좌/티커/월 변경 시 이전 행과 신규 행 각각 처리
- `DELETE_DIVIDEND`
  1. 이전 데이터 기준 월 셀 차감

## 7) 데이터 형식/반올림
- 월별 셀 값은 숫자형으로 저장
- 소수점은 시트 운영 방식에 맞춰 `소수 둘째 자리` 반올림 권장
- 음수 방지: 삭제/수정 차감 후 음수가 되면 `0`으로 보정(선택 정책)

## 8) 안정성 규칙
- 멱등성 키 권장: `dividend_id + action + updated_at`
- 재시도 시 중복반영 방지를 위해 로그 시트(`SYNC_LOG`) 또는 앱 DB에 처리 이력 저장
- 시트 쓰기 전 검증:
  - 대상 컬럼이 `K~V`인지 확인
  - 행 매칭이 정확한지 확인

## 9) 예시
- 예: `2026-03-15`, `토스증권`, `JEPQ`, `10.00`
  - 매칭 행: `B=토스증권`, `E=JEPQ`
  - 반영 컬럼: `M(3월)`
  - 동작: `M = M + 10.00`
- 이때 `W/X/Y/Z`는 시트 수식이 자동 계산

## 10) 구현 범위 제안
- 1차:
  - 생성(Create) 이벤트만 월 컬럼 가산
  - 수식 컬럼 보호 검증
- 2차:
  - 수정(Update) / 삭제(Delete) 역반영
  - 멱등성/재시도/오류 알림(로그)
